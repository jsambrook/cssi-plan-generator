# Mafia Offer Methodology

Source: Eli Goldratt's concept, operationalized as a four-phase consulting process in the Common Sense Systems Mafia Offer skill (`~/git/cssi-ai-share/skills/mafia-offer/`).

---

## What Is a Mafia Offer?

An offer so good the customer cannot refuse it. The term comes from Goldratt's observation that if you truly solve a customer's core problem — the structural conflict at the root of their industry's dysfunction — the offer becomes compelling because it eliminates pain the customer has accepted as normal.

A mafia offer is NOT:
- A discount or price reduction
- A bundle of extra features
- A risk-free trial

A mafia offer IS:
- A structural change in how the service/product is delivered
- Backed by specific guarantees that the provider can make because the injection genuinely works
- Targeted at an industry segment (not a specific prospect), so it scales

---

## The Four Phases

### Phase 1: Industry Perception Analysis

Research how customers perceive the target industry. Find ~8 UDEs (Undesirable Effects) — concrete, sourced, widespread complaints stated from the customer's perspective.

**Search strategy**: Reddit complaints, case studies, regulatory findings, lawsuit records, analyst reports, "waste of money" / "never again" language.

**Quality bar**: Each UDE must be specific (not vague), sourced (traceable), widespread (recurring theme), and customer-perspective (what buyers experience).

**Output**: 5-7 page document shared with prospects to validate. Framed as industry analysis, never about the specific prospect's company.

### Phase 2: Cause-Effect Analysis

Trace validated UDEs to root causes through cause-effect logic. Repeatedly ask "what causes this?" with sufficiency testing ("if cause, then effect" must read as logically true).

**Output**: 6-8 page document identifying 3-5 recurring root causes and a convergence point — the fundamental tension that all the UDEs trace back to.

### Phase 3: Core Conflict Analysis (Evaporating Cloud)

Articulate the Evaporating Cloud — the fundamental dilemma at the root of the industry's problems. Document all five elements and the assumptions behind each arrow. Identify which assumption is most vulnerable to challenge.

**Output**: 6-8 page document. The cloud is presented as a hypothesis, not a verdict.

### Phase 4: The Mafia Offer

Design an injection that breaks the dilemma. Package it as an offer with specific guarantees. The offer must:

- Eliminate the conflict without negative consequences
- Not require the customer to take significant risk
- Be something the provider can actually deliver
- Create a win-win outcome

**Output**: 8-10 page document ending with: "Can you deliver on these guarantees today?" If not, offer managed improvement services.

---

## Key Design Principles

1. **Industry, not prospect**: All documents discuss the segment. This lets prospects engage honestly without feeling accused.
2. **Phased delivery with checkpoints**: Each phase stands alone. The user validates before the next phase begins.
3. **Evidence-based**: UDEs and logic chains are grounded in research, not assumptions.
4. **The offer is structural**: It changes how work is done, not how it is priced.

---

## TOC Terminology Quick Reference

- **UDE**: Observable negative outcome, stated as present-tense fact. Not a solution in disguise.
- **CRT**: Current Reality Tree — cause-effect diagram with necessity logic (AND/OR relationships).
- **EC**: Evaporating Cloud — diagram exposing a core conflict as a false dilemma via hidden assumptions.
- **Injection**: A change that invalidates an EC assumption. Must create win-win, not introduce new problems.
