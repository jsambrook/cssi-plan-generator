## Idea

The cssi-plan-generator is an auto-improving business idea refinement tool. This run applies the tool to itself.

### Design principle: front-load human input, then run autonomously

The methodology now follows a clear pattern:
1. **Seed phase (interactive):** The agent does web research and drafts proposed answers to the four clarifying questions. The user corrects, refines, or approves — not answering from scratch. The user confirms the seed before iteration begins.
2. **Iteration phase (autonomous):** The agent runs the full iteration budget without mid-run confirmations. Injections from ECs are selected and applied by the agent. Mafia-offer phases run sequentially without checkpoints. All decisions are documented in the variant files for post-run human review.
3. **Plateau handling (autonomous with auto-stop):** When 3 consecutive iterations produce no new current-best, the agent tries a disruptive move. If that also fails, the agent tries one more move. If 3 total consecutive non-improvements occur post-plateau, the agent stops and writes the report. The user can always resume with "keep going."
4. **Report phase (human review):** The report flags every injection applied, every checkpoint that would have been a pause point, and every scoring decision where the red team step or regression check caught something. The human reviews holistically rather than being interrupted mid-flow.

## Red team (pre-scoring criticisms)

1. The "draft-and-correct" redesign for seed shaping (#2) could backfire: if the agent's drafted answers are wrong, the user may anchor on them rather than correcting. This could produce worse seeds than asking open questions.
2. Auto-stop after 3 consecutive non-improvements is a heuristic, not a principle. Some ideas need 5+ "failed" iterations before a breakthrough. The auto-stop might cut runs short.
3. Deferring injection validation to the report (#6, #7) means the agent might build 5 iterations on top of a bad injection before the human catches it. The compounding error risk is real.
