# Variant v007

## Move applied
Move 3: Adjust scope (narrow)

## Move rationale
v006 introduced a valuable idea (cross-run learning loop) that lost on tiebreak because it dropped Difficulty from 72 to 70. The cross-run scan + move-history.md were too much mechanism. Narrowing to the simplest version that preserves the value without the complexity cost.

## What changed

v006 proposed two mechanisms:
1. Cross-run `results.tsv` scan at run start
2. `runs/move-history.md` maintained across runs

Both require the agent to reliably perform cross-directory reads and file maintenance. The simpler version:

**Just read the existing results.tsv files.** No new file. No maintenance burden. The agent already knows how to read TSV files and already scans the current run's `results.tsv` before each move selection (per `references/select.md`). Extending this to read `runs/*/results.tsv` is a one-line conceptual change, not a new system.

The `move-history.md` idea is dropped. The narrative meta-knowledge it would have captured is available in the iteration logs and cross-session report — which already exist. Adding a third narrative file creates redundancy and maintenance burden.

## Idea

The cssi-plan-generator is an auto-improving business idea refinement tool that gets better with use.

### Cross-run learning (simplified from v006)

At the start of each new run, the agent reads all existing `runs/*/results.tsv` files alongside the current run's history. This gives move selection a prior: "Move 5 has produced new current-bests in 3 of 4 past applications; Move 8 has never won." The prior is advisory — different ideas have different needs — but it's better than starting cold.

No new files. No maintenance. Just a wider read scope for the existing move selection heuristic.

### Design principle: front-load human input, then run autonomously
1. Seed phase (interactive, draft-and-correct)
2. Iteration phase (autonomous, no mid-run confirmations)
3. Plateau handling (auto-stop after 3 consecutive non-improvements)
4. Report phase (human review)

## Red team (pre-scoring criticisms)

1. "Just read the TSV files" sounds trivial, but the agent must actually do it reliably every run. There's no enforcement mechanism — it's an instruction in CLAUDE.md that the agent might skip.
2. The advisory framing ("prior, not rule") is good in theory, but in practice an LLM reading "Move 5 won 3 of 4 times" will likely over-weight that signal. The overfitting risk from v006 is reduced but not eliminated.
3. Dropping `move-history.md` means losing the narrative meta-knowledge (e.g., "Move 5 works best when failure modes is the bottleneck"). The TSV data shows *what* happened but not *why*. However, this is the right trade-off for now — simplicity over completeness.
